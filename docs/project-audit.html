1. 项目愿景

Paporot 是一个 AI 生成代码的行为审计与沙盒化分析管道。它回答两个问题：

  • Agent 这次提交改了什么能力？（6-Skill DAG 自动编排分析）
  • Agent 的行为变好还是变坏了？（Contract → Evidence → Replay 三层验证）

2. 用户实际入口

当前 CLI 路由机制： init / replay → Native 直接执行；其余 → wasmtime 沙盒。沙盒只认 analyze 和 skill list，其他 16 个返回 Unknown command。

4. 架构对比

图 2：旧架构 —— Agent 三层漏斗（16 条命令，全部不可达）
图 3：新架构 —— WASM Skill DAG 沙盒管线（4 个可用命令）
关键区别：旧架构在 Native 进程内直接执行 L1→L2→L3 漏斗，新架构把所有分析逻辑隔离在 paporot-core.wasm 沙盒内，通过 6 个 Skill + DAG 调度执行，只通过 3 个受控 host function 与外部交互。

6. 重构路线图

第一步：清洗死代码 CLI
删除 src/commands/ 下 16 个不可达命令文件、src/agent.rs、src/storage.rs，精简 src/cli.rs 到只保留 4 个可用命令。低风险、高价值。
第二步：添加 paporot skill run <name>
允许用户单独运行某个 Skill，自动补齐 DAG 依赖链。修改 crates/paporot-core/src/main.rs 的 skill 子命令。
第三步：补 MVP 功能坑
prd-coverage Skill → 编译 .wasm；HTML syntax checker；conforms_to_schema 真校验。
第四步：合并重复的 Skill 执行逻辑
统一 pipeline.rs 与 verification_runner.rs 的重复代码。
第五步：清理警告 + 修复文档引用
cargo fix 解决 23 个 warning，补全 docs/ 目录的 PRD 引用。

7. 建议的最终 CLI 形态

paporot init paporot analyze [--prd <p>] paporot skill list paporot skill run <name> paporot replay list paporot replay show <id> paporot replay stats paporot replay run --from --to

8 个命令，每个都可运行。analyze 末尾自动跑 Replay 并输出回归摘要。
