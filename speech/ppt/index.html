Data Source · Trace
Act I · 4 / 14
所有分析的起点

Execution Trace

一次 Agent 执行的完整记录：prompt、按时间排序的 tool 调用序列、每次调用的参数和耗时、观察结果、最终输出、token 消耗。

Trace 怎么来？SDK 自动捕获（Captured）或日志导入（Imported）。Paporot 不做运行时监控，只消费已产生的 Trace 数据。 BehaviorTrace { id, session_id, tool_calls, observations, token_usage, ... } 
BehaviorTrace {
  id: "trace_20260619_001"
  session_id: "sess_abc123"
  prompt: "修复登录页面的 token 过期问题"
  tool_calls: [
    { tool_name: "read",
      args: {"path": "src/auth.ts"},
      duration_ms: 145 },
    { tool_name: "edit",
      args: {"path": "src/auth.ts"},
      duration_ms: 230 },
    ...
  ]
  observations: [...]
  token_usage: { input: 3400, output: 1200 }
  source: Captured { agent_name: "claude-code" }
}

Act III · 7 / 14

P0 · 行为状态机

把 Agent 的线性 tool call 序列分段：定位 → 修改 → 验证 → 提交。三层 pipeline 自动构建。

Layer 1 · 硬分割

RuleSegmenter 按 tool 类别硬切：read/grep → locate，write/edit → modify，test/build → verify，commit → commit

Layer 2 · 窗口候选

WindowBuilder 滑动窗口找细粒度状态候选，每个窗口一个主导 phase 标签

Layer 3 · 相邻合并

AdjacentMerger 合并相似候选状态 → 构建 BehaviorStateGraph

P0 · BehaviorStateGraph · RuleSegmenter → WindowBuilder → AdjacentMerger

P1 · 10 维轨迹向量

TrajectoryVector: 将一次执行映射到可比较的数值向量空间。D7 规范化 pipeline，禁止 whitening（保持语义轴可解释性）。

分布类维度
tool_distribution — 工具类别占比
state_distribution — 状态分布
Agent 花多少时间定位 vs 修改？
熵类维度
tool_entropy — 工具序列混乱度
phase_entropy — 阶段转换不可预测性
transition_entropy — 状态转移随机性
模式 + 稳定性
loop_ratio — 循环占比
backtrack_ratio — 回溯比例

Behavior · P2 耦合
Act III · 9 / 14
改了 A，B 和 C 会受影响吗？

P2 · 行为耦合图

从 Agent 实际行为中自动推导 capability 之间的耦合关系。不是理论上的"耦合度"，而是真实的行为联动。

D9 核心公式
correlation = cochange × (1 + λ × similarity)
cochange: 3 层 log-saturated 证据
similarity: P1 向量余弦相似度
λ ∈ [0.2, 0.4]: 相似度调制因子
4 层存活过滤 (hard → purity → stability → top-K) 剪枝，只保留最稳定的耦合边。Auth 模块高频牵连 Payment + Dashboard + Inventory → 架构瓶颈，改它要小心。 CouplingBuilder · D9 Formula · 4-Layer Survivorship Filter

Security · 安全模型
Act III · 11 / 14
分析 AI Agent 的工具不能本身不安全

三层安全模型

1
WASM 沙盒硬隔离

paporot-core.wasm 编译到 wasm32-wasip1。无法直接访问文件系统、网络、系统调用。

2
3 个受控 Host Func

host_read_file · host_write_file · host_llm_call —— 宿主层做访问控制和审计。

3
零持久化沙盒内状态

每次 paporot analyze 从零加载 wasm，不存在跨执行状态泄露。

如果你用一个不安全的工具去审计另一个 Agent，这本身就是一个笑话。 Zero Trust · WASM Isolation · Controlled I/O
Security · WASM Hard Isolation + 3 Host Functions + Zero Persistent State
